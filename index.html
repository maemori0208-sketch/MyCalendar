      繰り返し予定

      この操作の適用範囲を選択してください。

      自動同期（クラウド）

      Googleアカウントでログインし、データをあなたのGoogleドライブ（アプリ専用の隠しフォルダ）に保存して自動同期します。Firebase不要・データベース構築も不要。データはあなたのDrive内にあり、他アプリからは見えません。
      初回のみ、GoogleのOAuthクライアントID（無料）が必要です。

      ✓ 接続中
      初期設定の手順（初回のみ・約5分・無料）
      1. Google Cloud Console でプロジェクトを作成 →「APIとサービス → ライブラリ」で Google Drive API を有効化
      2. 「OAuth 同意画面」を構成（User Type=外部）。「テストユーザー」に自分のGmailを追加（appdataのみのため公開審査は不要）
      3. 「認証情報 → 認証情報を作成 → OAuth クライアントID → ウェブアプリケーション」を作成。承認済みのJavaScript生成元に公開URL（例 https://xxxx.netlify.app）と http://localhost を追加
      4. 発行されたクライアントID（…apps.googleusercontent.com）を上に貼り付けて「Googleでログイン」
      5. もう一方の端末でも同じクライアントIDでログインすれば同期されます

      Googleアカウントでログインすると、あなた専用の領域に自動同期されます（本人のみ読み書き可）。各端末で「Googleでログイン」するだけです。
      初回のみ、無料のFirebaseプロジェクトの設定貼り付けが必要です。

      ✓ ログイン中
      初期設定の手順（初回のみ・約5分・無料）
      1. Firebase コンソールでプロジェクトを作成
      2. 「構築 → Realtime Database」→「データベースを作成」。ルールを次のように設定：
        { "rules": { "users": { "$u": { ".read": "auth!=null&&auth.uid==$u", ".write": "auth!=null&&auth.uid==$u" } } } }
      3. 「構築 → Authentication → ログイン方法」で Google を有効化
      4. 「Authentication → 設定 → 承認済みドメイン」に公開URL（例：xxxx.netlify.app）を追加
      5. 「プロジェクトの設定 → マイアプリ（ウェブ）」の 構成（firebaseConfig）をコピーして上の欄に貼り付け
      6. 「Googleでログイン」。もう一方の端末でも同じ設定を貼り付けてログインすれば同期されます

      ログイン不要。両端末で同じ「データベースURL」と「同期コード」を設定します。同期コードを知る人はデータにアクセスできるため、長めの合言葉にしてください。

      Googleカレンダー取り込み

      Googleカレンダーの予定をこのアプリに表示します（読み取り専用）。あなたのGoogleカレンダーには一切書き込みません。予定は「G」印付きで表示され、タップするとGoogleカレンダーで開きます。

      ✓ 接続中 Googleカレンダー

      双方向：未接続
      初期設定の手順（初回のみ・約5分・無料）
      1. Google Cloud Console でプロジェクトを作成 →「APIとサービス → ライブラリ」で Google Calendar API を有効化
      2. 「OAuth 同意画面」を構成（User Type=外部）。「テストユーザー」に自分のGmailを追加
      3. 「認証情報 → OAuth クライアントID → ウェブアプリケーション」を作成。承認済みのJavaScript生成元に公開URL（例 https://xxxx.netlify.app）と http://localhost を追加
      4. 発行されたクライアントIDを上に貼り付けて「Googleでログイン」

      ※ Googleドライブ同期と同じOAuthクライアントIDを使えます（Calendar APIの有効化は別途必要）。
